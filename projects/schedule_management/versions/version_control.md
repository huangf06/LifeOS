# 时间计划版本管理系统

## 版本命名规则
- **主版本号.次版本号.修订号** (如: 1.0.0)
- 主版本号: 整体架构重大变更
- 次版本号: 时间块调整、新增/删除活动
- 修订号: 细节优化、参数调整

## 当前版本历史
- **v1.0.0** (2025-08-30): 基于认知科学的初版超级执行机器时间表

## 版本管理流程

### 1. 创建新版本
当需要调整计划时：
```bash
# 复制当前版本作为新版本基础
cp plans/master_schedule_v1.0.json plans/master_schedule_v1.1.json
# 编辑新版本
# 更新version字段和created_date
# 记录变更原因
```

### 2. 变更记录
每次版本更新必须记录：
- 变更原因 (性能问题/生活变化/效果评估)
- 具体变更内容
- 预期效果
- 回滚条件

### 3. A/B测试机制
- 新版本试行7天
- 记录关键指标对比
- 根据数据决定是否保留
- 不良版本立即回滚

## 动态调整触发条件

### 自动触发调整
- 连续3天完成率<70% → 降低强度10%
- 连续7天某时段效率<60% → 重新分配该时段
- 月度目标偏差>30% → 全面策略调整

### 手动触发调整
- 生活状况变化 (面试密集期/身体状况)
- 新的学习需求
- 时间冲突
- 效果不达预期

### 季节性调整
- 春季: 增加户外活动比例
- 夏季: 早起时间提前30分钟
- 秋季: 调整为应试冲刺模式
- 冬季: 增加室内恢复时间

## 版本评估标准

### 量化指标
1. **执行完成率**: 每日时间块完成百分比
2. **认知效率**: 学习内容掌握速度
3. **能量水平**: 每日精力状态评分
4. **目标进度**: 三大任务月度完成情况

### 定性指标
1. **可持续性**: 是否感到过度疲劳
2. **生活平衡**: 是否有足够恢复时间
3. **心理状态**: 动力和满意度
4. **适应性**: 面对变化的灵活性

## 应急调整预案

### 面试高峰期 (v1.1-interview)
- 语言训练聚焦英语面试
- 专业学习增加2小时
- 健身压缩至60分钟
- 持续时间: 面试前2周

### 身体疲劳期 (v1.1-recovery)
- 所有高负荷时段减少30%
- 增加恢复时间50%
- 重点保持最低训练量
- 持续时间: 恢复至正常状态

### 高效爆发期 (v1.1-boost)
- 所有训练强度提升20%
- 减少恢复时间10%
- 记录高效状态触发因素
- 持续时间: 最多连续14天

## 版本切换SOP

1. **备份当前数据**
   - 导出过去7天执行记录
   - 保存当前版本配置

2. **新版本部署**
   - 更新时间表配置
   - 调整监控参数
   - 设置新的提醒

3. **过渡期管理**
   - 前3天密切监控
   - 每日记录适应情况
   - 必要时微调参数

4. **效果评估**
   - 7天后进行第一次评估
   - 30天后进行全面评估
   - 决定是否永久采用
